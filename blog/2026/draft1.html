Convex Hull Complexity Lower Bound

From undergraduate algorithms course, we learned that sorting $n$ elements in the comparison model requires $\Omega(n\log n)$ comparisons. Later, we learned that using this lower bound, we can prove an $\Omega(n\log n)$ lower bound for the convex hull problem in computational geometry, where given an unordered set of $n$ points in $\RR^2$, construct the convex hull of this set of points.

This is often proved by reducing from the sorting problem. By mapping $x_i\mapsto (x_i,x_i^2)\in \RR^2$, the convex hull must return the list in some cyclic shift of the sorted order. Therefore any convex hull algorithm must have $\Omega(n\log n)$ time complexity. This proof was not satisfactory to me, because it does not work if we only want to determine the vertices on the convex hull.

Consider the weaker problem of computing the number of points on the convex hull.
